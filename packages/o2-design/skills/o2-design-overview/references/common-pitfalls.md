# 常见坑与规避建议

## 1. 响应式丢失

现象: 给 reactive 对象动态新增属性后, 视图不更新

原因: O2 Design 的响应式底层是 Vue2, 动态新增属性不是响应式

规避:

- 在初始化时声明完整字段结构
- 或者使用可追踪的替换式赋值

## 2. 大页面输入卡顿

现象: 修改单个字段导致整页重渲染, 输入有明显卡顿

原因: 一个大型 render 函数绑定了大量响应式依赖

规避:

- 用 computed 将页面拆成多个子块
- 将大块区域拆成子组件, 缩小渲染影响面
- 列表与表单混合页优先做分区懒加载

## 3. 表格动态列异常

现象: 动态增删列后校验/required/列状态不符合预期

原因: table option 与 column 实例缓存未同步

规避:

- 动态列变更后, 走统一列重建流程
- 避免在渲染函数中频繁重建 columns 数组

## 4. 异步流程导致状态错乱

现象: 快速点击触发多次请求, 或者页面状态被旧请求覆盖

规避:

- 操作按钮使用 asyncHandler
- 关键异步流程添加请求序号或取消机制

## 5. Hooks 使用位置错误

现象: 生命周期或组合函数不生效

原因: 相关 API 只能在 setup 内调用

规避:

- 所有 useXxx/onMounted/onUpdated 统一放在 setup 顶层
- 不在 render 分支或普通工具函数中直接调用生命周期 API
